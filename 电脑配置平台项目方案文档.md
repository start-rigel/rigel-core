# 电脑配置平台项目方案文档（业务说明）

## 1. 项目现在要做什么

当前项目只做一件核心事情：

基于京东联盟获取到的硬件价格信息，结合用户需求，请求 AI 返回一套电脑配置推荐结果。

一句话：

`京东价格数据 + 用户需求 -> AI 推荐配置`

## 2. 项目核心逻辑

这个项目的核心，不是抓取本身，也不是前端页面本身。

当前真正的核心逻辑是：

1. 获取原始硬件商品和价格
2. 按天把原始商品整理成型号级价格快照
3. 把用户需求和价格清单一起交给 AI
4. 输出一份结构化、可解释的装机推荐结果

核心能力可以收成一句话：

`把杂乱的硬件价格数据，整理成可用的装机决策结果`

对产品价值来说，最关键的不是“搜到了多少商品”，而是：

- 价格是否可信
- 型号是否整理清楚
- 推荐结果是否让用户看得懂、敢参考

## 3. 当前主链路

1. 页面维护型号词库，支持 Excel 导入导出
2. `rigel-jd-collector` 按配置时间调用京东联盟接口获取商品与价格
3. 原始商品、价格快照和每日型号价格快照写入数据库
4. `rigel-build-engine` 读取数据库中的当前价格数据并请求 AI API
5. `rigel-console` 展示结果

## 4. 当前模块

### 4.1 `rigel-jd-collector`

职责：

- 读取型号词库
- 调用京东联盟接口
- 搜索电脑硬件商品
- 保存原始商品和价格快照
- 生成每日型号价格快照

### 4.2 `rigel-build-engine`

职责：

- 接收预算、用途、偏好等用户参数
- 读取数据库中的当前价格数据
- 组织 AI 输入
- 请求 AI API
- 返回结构化推荐结果

### 4.3 `rigel-console`

职责：

- 提供最小页面与 API 入口
- 提供型号词库管理页面
- 将用户请求转发到 `rigel-build-engine`
- 展示推荐结果

## 5. 业务关键点

### 5.1 型号词库维护

当前型号词库统一采用：

- 页面手动维护
- Excel 批量导入
- Excel 批量导出

当前不使用 CSV 和 JSON 作为业务侧批量导入格式。

### 5.2 京东数据来源

当前正式使用的京东联盟接口：

1. `jd.union.open.goods.query`
2. `jd.union.open.goods.promotiongoodsinfo.query`
3. `jd.union.open.category.goods.get`

未来返佣链接预留：

4. `jd.union.open.promotion.common.get`

### 5.3 AI 输入与输出

AI 输入当前分成两层：

- `console -> build-engine`：
  - 顶层用户需求字段
  - `catalog.items` 型号级价格目录
- `build-engine -> AI`：
  - `user_request`
  - `price_catalog`

AI 输出固定为结构化推荐结果，至少包含：

- `summary`
- `parts`
- `total_price`
- `reasoning`
- `alternatives`
- `warnings`

## 6. 当前暂不做

当前明确不做：

- 闲鱼
- 浏览器抓京东
- 复杂规则引擎
- 重型后台系统
- 多平台联合推荐
- 复杂规格参数库

## 7. 当前最小可交付

当前最小可交付版本只要求能稳定完成：

1. Excel 导入型号词库
2. 每日定时采集京东商品与价格
3. 形成每日型号级价格快照
4. 接收用户预算和用途
5. 请求 AI 获取结构化推荐结果
6. 页面展示推荐结果
