# 电脑配置平台项目方案文档（重整理版）

## 1. 项目现在要做什么

当前项目只做一件核心事情：

基于京东联盟获取到的硬件价格信息，结合用户需求，请求 AI 返回一套电脑配置推荐结果。

一句话：

`京东价格数据 + 用户需求 -> AI 推荐配置`

## 2. 当前主链路

1. `rigel-jd-collector` 调用京东联盟接口获取商品与价格
2. 原始商品与价格写入数据库
3. `rigel-build-engine` 读取原始商品并整理成型号级价格清单
4. `rigel-build-engine` 接收用户需求并请求 AI API
5. `rigel-console` 展示结果

## 3. 当前模块

### 3.1 `rigel-jd-collector`

职责：

- 调用京东联盟接口
- 搜索电脑硬件商品
- 保存原始商品和价格快照

### 3.2 `rigel-build-engine`

职责：

- 接收预算、用途、偏好等用户参数
- 整理当前硬件信息
- 形成型号级价格清单
- 组织 AI 输入
- 请求 AI API
- 返回结构化推荐结果

### 3.3 `rigel-console`

职责：

- 提供最小页面与 API 入口
- 将用户请求转发到 `rigel-build-engine`
- 展示推荐结果

## 4. AI 输入规范

AI 输入固定由两部分组成：

- `user_request`
- `price_catalog`

### 4.1 `user_request`

包含：

- `budget`
- `usage`
- `brand_preference`
- `special_requirements`
- `notes`

### 4.2 `price_catalog`

按类别分组：

- `cpu`
- `gpu`
- `motherboard`
- `ram`
- `ssd`
- `psu`
- `case`
- `cooler`

每个型号对象包含：

- `model`
- `price`
- `price_min`
- `price_max`
- `sample_count`

## 5. AI 输出规范

AI 输出必须是结构化 JSON，至少包含：

- `summary`
- `parts`
- `total_price`
- `reasoning`
- `alternatives`
- `warnings`

`parts` 每项必须包含：

- `category`
- `model`
- `price`
- `reason`

## 6. 当前暂不做

当前明确不做：

- 闲鱼
- 浏览器抓京东
- 复杂规则引擎
- 重型后台系统
- 多平台联合推荐
- 复杂规格参数库

## 7. 当前最小可交付

当前最小可交付版本只要求能稳定完成：

1. 采集京东商品与价格
2. 形成型号级价格清单
3. 接收用户预算和用途
4. 请求 AI 获取结构化推荐结果
5. 页面展示推荐结果
