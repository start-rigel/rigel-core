# 电脑配置平台项目方案文档（业务说明）

## 1. 项目现在要做什么

当前项目只做一件核心事情：

基于京东联盟获取到的硬件价格信息，结合用户需求，请求 AI 返回一套电脑配置推荐结果。

一句话：

`京东价格数据 + 用户需求 -> AI 推荐配置`

## 2. 当前主链路

1. 页面维护型号词库，支持 Excel 导入导出
2. `rigel-jd-collector` 调用京东联盟接口获取商品与价格
3. 原始商品与价格写入数据库
4. `rigel-build-engine` 读取原始商品并整理成型号级价格清单
5. `rigel-build-engine` 接收用户需求并请求 AI API
6. `rigel-console` 展示结果

## 3. 当前模块

### 3.1 `rigel-jd-collector`

职责：

- 读取型号词库
- 调用京东联盟接口
- 搜索电脑硬件商品
- 保存原始商品和价格快照

### 3.2 `rigel-build-engine`

职责：

- 接收预算、用途、偏好等用户参数
- 整理当前硬件信息
- 形成型号级价格清单
- 组织 AI 输入
- 请求 AI API
- 返回结构化推荐结果

### 3.3 `rigel-console`

职责：

- 提供最小页面与 API 入口
- 提供型号词库管理页面
- 将用户请求转发到 `rigel-build-engine`
- 展示推荐结果

## 4. 业务关键点

### 4.1 型号词库维护

当前型号词库统一采用：

- 页面手动维护
- Excel 批量导入
- Excel 批量导出

当前不使用 CSV 和 JSON 作为业务侧批量导入格式。

### 4.2 京东数据来源

当前正式使用的京东联盟接口：

1. `jd.union.open.goods.query`
2. `jd.union.open.goods.promotiongoodsinfo.query`
3. `jd.union.open.category.goods.get`

未来返佣链接预留：

4. `jd.union.open.promotion.common.get`

### 4.3 AI 输入与输出

AI 输入固定由两部分组成：

- 顶层用户需求字段
- `catalog.items` 型号级价格目录

AI 输出固定为结构化推荐结果，至少包含：

- `summary`
- `parts`
- `total_price`
- `reasoning`
- `alternatives`
- `warnings`

## 5. 当前暂不做

当前明确不做：

- 闲鱼
- 浏览器抓京东
- 复杂规则引擎
- 重型后台系统
- 多平台联合推荐
- 复杂规格参数库

## 6. 当前最小可交付

当前最小可交付版本只要求能稳定完成：

1. Excel 导入型号词库
2. 采集京东商品与价格
3. 形成型号级价格清单
4. 接收用户预算和用途
5. 请求 AI 获取结构化推荐结果
6. 页面展示推荐结果
