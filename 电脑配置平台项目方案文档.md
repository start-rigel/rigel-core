# 电脑配置平台项目方案文档（当前范围）

## 1. 当前目标

当前阶段只保留一条主链路：

1. 采集京东电脑配件价格
2. 将原始商品标题标准化为 canonical model
3. 生成按型号聚合的价格目录
4. 由 `rigel-build-engine` 基于 `预算 + 用途 + 当前价格目录` 生成推荐结果
5. 输出可读的配置建议

当前范围不包含已移除模块。

## 2. 当前激活模块

| 中文名称 | 英文名称 | repo / 服务名 | 说明 |
|---|---|---|---|
| Rigel-京东采集模块 | JD Collector | `rigel-jd-collector` | 负责京东新品配件价格采集 |
| Rigel-配置引擎模块 | Build Engine | `rigel-build-engine` | 负责标准化、价格聚合、推荐生成和最小兼容校验 |
| Rigel-应用模块 | Console | `rigel-console` | 负责前台、后台和 API 聚合 |

## 3. 当前边界

### 3.1 京东采集模块

负责：
- 按关键词搜索京东商品
- 记录标题、SKU、价格、店铺、链接等原始信息
- 追加价格快照
- 提供给后续标准化与聚合使用

不负责：
- 兼容性决策
- AI 推荐
- 前台展示

### 3.2 配置引擎模块

负责：
- 标准化原始标题
- 映射 canonical model
- 聚合价格目录
- 保留最小硬校验

不负责：
- 直接抓取外部平台
- 直接承担前台页面
- 替代 AI 输出文案

### 3.3 应用模块

负责：
- 聚合 build-engine
- 提供最小前台和后台入口

不负责：
- 直接读数据库做核心业务
- 直接做规则决策

## 4. 当前 MVP 输出

当前 MVP 要稳定回答的问题是：

- 给定 `6000 元` 和 `打游戏`，系统能否基于当前京东价格目录给出一套可读推荐？

## 5. 当前暂不做

- 已移除模块接入
- 二手参考价
- 多平台联合推荐
- 复杂兼容规则系统
- 重后台运营系统

## 6. 当前文档约束

当前方案文档只描述激活范围。
已暂时移除的模块不在本文件中继续展开，以免和当前实现方向冲突。
